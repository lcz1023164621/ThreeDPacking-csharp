# Elite CS612 扫码装箱及暂存重写方案

## 现有问题分析

当前失败程序的主流程是数字标志位驱动：

- `var_1 == 8`：当前抓取物放暂存台。
- `var_1 == 6 or 7`：当前抓取物直接装箱。
- `var_9 == 9`：装箱后再从暂存台取物。
- `Server(发送数据:=11)`：动作完成后通知上位机。

这套程序在“抓取顺序 BAC、放置顺序 ACB”时容易漏取 B，核心原因有四个：

1. `9` 和 `6` 是两个连续数字信号，但 TCP 不是消息队列。上位机连续发送时，机械臂可能一次收到 `9\n6\n`。如果条件判断没有先做 `split("\n")[0].strip()`，`var_9 ?= 9` 就可能不成立；如果做了切分，后面的 `6` 又可能被当成下一次接收的 `buffer_str` 或被丢掉。
2. “取暂存物”和“装箱方式”被拆成两个信号：先 `9`，再 `6/7`。这两个信号实际属于同一个动作，拆开发送会导致顺序错读。
3. `buffer_seq` 同时被用作暂存槽序号和装箱位序号。抓取顺序和放置顺序不一致时，暂存槽号不等于装箱步骤号，`get_position_by_index(下放坐标, buffer_seq_1-1)` 可能取错装箱坐标。
4. 回取暂存只藏在某些分支里，而不是主状态机的独立动作。放完 C 后应该由上位机明确下发“从暂存台取 B 并放到 B 的装箱位”，机械臂不应该靠 `9` 后再猜 `6` 的含义。

## 新通信协议

所有消息都用字符串，单条命令以 `\n` 结束。不要再连续发送 `9\n6\n` 这种拆分标志位。

### 双 Socket 约定

既然 Elite 控制器可以配置多个 Socket，建议直接分成两个通道：

| Socket | 方向 | 建议端口 | 数据类型 | 用途 |
| --- | --- | --- | --- | --- |
| `socket1` | PC -> Robot | `10000` | 字符串 | 机械臂接收上位机命令 |
| `socket2` | Robot -> PC | `10001` | 字符串 | 机械臂发送事件/完成信号 |

下面程序里用 `Server_RX(...)` 表示选择 `socket1` 的接收节点，用 `Server_TX(...)` 表示选择 `socket2` 的发送节点。示教器里如果不能直接改函数名，就仍然使用 `Server(...)`，但每个节点要选择对应的 Socket。

### 上位机发给机械臂

| 命令 | 含义 |
| --- | --- |
| `CMD_PICK_SCAN` | 从来料区抓取下一个物体并拍照/扫码 |
| `CMD_PLACE_HELD|item=A|place=1|pose=BOX_ORIGIN` | 将当前夹持物直接装箱 |
| `CMD_PLACE_HELD|item=A|place=1|pose=TEST_BOX` | 将当前夹持物按测放装箱位装箱 |
| `CMD_BUFFER_HELD|item=B|slot=1` | 将当前夹持物放到暂存台指定槽位 |
| `CMD_TAKE_BUFFER|item=B|slot=1|place=3|pose=BOX_ORIGIN` | 从暂存台取物并装箱 |
| `CMD_DONE` | 全部完成 |
| `CMD_ABORT|reason=xxx` | 异常中止 |

关键要求：`CMD_TAKE_BUFFER` 必须同时带 `slot`、`place`、`pose`，不能拆成 `9` 和 `6` 两次发送。

### 机械臂发给上位机

| 事件 | 含义 |
| --- | --- |
| `EVT_ROBOT_READY` | 机械臂已启动，等待命令 |
| `EVT_SCAN_READY|pick=0` | 已抓取并到达拍照位，等待上位机识别结果/调度命令 |
| `EVT_HELD_PLACED|item=A|place=1` | 当前夹持物已装箱 |
| `EVT_HELD_BUFFERED|item=B|slot=1` | 当前夹持物已放入暂存台 |
| `EVT_BUFFER_PLACED|item=B|slot=1|place=3` | 暂存物已取回并装箱 |
| `EVT_ACTION_DONE` | 当前动作完成，可下发下一条命令 |
| `EVT_DONE` | 程序结束 |
| `EVT_ERROR|reason=xxx` | 机械臂侧异常 |

拍照旋转子流程也使用字符串：

- 原 `Server(发送数据:=0)` 改为 `Server_TX(发送数据:="EVT_PHOTO_AT_POSE")`
- 原 `Server(发送数据:=5)` 改为 `Server_TX(发送数据:="EVT_VISION_WAIT")`
- 原接收 `issucceed` 的节点使用 `socket1`：`Server_RX(issucceed:=接收到的值)`
- 原接收 `issucceed` 中的 `"1"`/`"2"` 改为接收 `VISION_OK` / `VISION_ROTATE` / `VISION_FAIL`

## 需要保留或新增的脚本函数

继续保留已有脚本：

- `strtoshuju.script`
- `strtobuffer.script`

建议替换或扩展 `parse_action.script`，增加下面这些函数：

```python
def first_line(raw):
    if raw == None:
        return ""
    return str(raw).split("\n")[0].strip()

def get_action(cmd):
    return first_line(cmd).split("|")[0].strip()

def get_field(cmd, key):
    parts = first_line(cmd).split("|")
    for p in parts[1:]:
        kv = p.split("=", 1)
        if len(kv) == 2 and kv[0].strip() == key:
            return kv[1].strip()
    return ""

def get_int_field(cmd, key, default_value):
    v = get_field(cmd, key)
    if v == "":
        return default_value
    return int(v)
```

## BAC 抓取、ACB 放置时的正确通信序列

```text
Robot -> PC: EVT_ROBOT_READY

PC -> Robot: CMD_PICK_SCAN
Robot 抓到 B 并拍照
Robot -> PC: EVT_SCAN_READY|pick=0
PC -> Robot: CMD_BUFFER_HELD|item=B|slot=1
Robot -> PC: EVT_HELD_BUFFERED|item=B|slot=1
Robot -> PC: EVT_ACTION_DONE

PC -> Robot: CMD_PICK_SCAN
Robot 抓到 A 并拍照
Robot -> PC: EVT_SCAN_READY|pick=1
PC -> Robot: CMD_PLACE_HELD|item=A|place=1|pose=BOX_ORIGIN
Robot -> PC: EVT_HELD_PLACED|item=A|place=1
Robot -> PC: EVT_ACTION_DONE

PC -> Robot: CMD_PICK_SCAN
Robot 抓到 C 并拍照
Robot -> PC: EVT_SCAN_READY|pick=2
PC -> Robot: CMD_PLACE_HELD|item=C|place=2|pose=BOX_ORIGIN
Robot -> PC: EVT_HELD_PLACED|item=C|place=2
Robot -> PC: EVT_ACTION_DONE

PC -> Robot: CMD_TAKE_BUFFER|item=B|slot=1|place=3|pose=BOX_ORIGIN
Robot 从暂存台取 B 并装箱
Robot -> PC: EVT_BUFFER_PLACED|item=B|slot=1|place=3
Robot -> PC: EVT_ACTION_DONE

PC -> Robot: CMD_DONE
Robot -> PC: EVT_DONE
```

这里不再出现 `9\n6\n`。回取 B 的暂存槽和 B 的装箱位在同一条命令里明确给出：`slot=1`、`place=3`。

## 上位机侧必须同步修改

1. 每条命令只发送一行，末尾加 `\n`。
2. 不要连续发送两个动作标志位；一个机械臂动作对应一条完整命令。
3. 从暂存台取物时发送 `CMD_TAKE_BUFFER|item=B|slot=1|place=3|pose=BOX_ORIGIN\n`。
4. 机械臂发 `EVT_ACTION_DONE` 后，上位机再发下一条命令。
5. PC 调度表里要分开保存：
   - `slot`：物体在哪个暂存槽。
   - `place`：物体在装箱顺序里的第几个位置。
6. 上位机发送命令只连/写 `socket1`；上位机监听/读取机械臂事件只用 `socket2`。
