# 示教器主装箱流程（8056 暂存 + 信号 8/9/10）

本文档对应 PC 端暂存台装箱协调逻辑，供机械臂示教器实现参考。变量名可按现场调整。

## 通信端口

| 服务 | 端口 | 说明 |
|------|------|------|
| 装箱坐标 | 8055 | 批次开始时 PC 一次性下发全部箱内下放点位 |
| 暂存坐标 | 8056 | 运行时逐条接收 `seq,下放点位` |
| PC→臂信号 | 10000 | 6/7/8/9/10 等动作信号 |
| 臂→PC信号 | 15000 | 0/3/4/5 等状态信号 |

## 坐标系与点位含义（装箱 / 暂存通用）

PC 下发的 `[x,y,z,0,0,0]` **不是机械臂基座坐标**，而是相对**示教坐标系原点**的偏移（数值通常较小）：

1. 示教器定义装箱 / 暂存各自的坐标系原点与方向，以及 `pickorigin`（原点上方的抓取过渡点）
2. 运行时先 `MoveJ pickorigin`（装箱系或暂存系）
3. 再按 PC 下发的偏移：**先 XY，再 Z**
4. 箱内最终放置仍用 8055 批次中的 `box_drop[seq]`

装箱点位（8055）与暂存点位（8056）**格式完全一致**，仅坐标系不同（装箱原点 vs 暂存原点）。

## 8056 消息格式

```
{装箱顺序},{抓取点位},{下放点位}
```

示例（顺序 2，300×80×40mm 长条平铺后顶面中心在 200mm, 40mm, 40mm）：

```
2,[0.2,0.04,0.04,0,0,0],[0.2,0.04,-0.42,0,0,0]
```

- 单位：米，与 8055 一致
- `pick` / `drop`：相对**暂存坐标系原点**的偏移；由 PC 按 **500×500mm 台面、物体尺寸平铺** 计算
- 示教器解析见仓库根目录 `strtobuffer.py`（`get_var_current_position` 供 `var_current_position` 使用）
- 从暂存取：同一偏移作为抓取点，箱内目标位仍用 `box_drop[seq]`

## 初始化

```
连接 PC
├─ Server 装箱坐标服务    端口 8055    // 批次开始接收全部箱内点位，存入 box_drop[1..N]
├─ Server 暂存坐标服务    端口 8056    // 运行时逐条接收 buffer_pick / buffer_drop
├─ Server 信号发送服务    端口 10000   // PC→臂 发 6/7/8/9/10
└─ Server 信号接收服务    端口 15000   // 臂→PC 发 0/3/4/5
```

## 主循环（未收到信号 4）

```
循环 未收到信号4（批次未结束）
│
├─ MoveJ  原料区_扫码路点
│
├─ Server(发送数据: 0)                    // 臂→PC：到位请求扫码
├─ Server(pc_signal := 接收到的值)        // 等待 PC 发 1 或 2
├─ 解析 pc_signal → var_signal            // 先判'1'再判'2'，避免缓存干扰
│
├─ 循环 var_signal != '1'                 // === 拍照旋转子流程（可复用现有）===
│  ├─ MoveL  换姿路点
│  ├─ Server(发送数据: 0)
│  ├─ Server(pc_signal := 接收到的值)
│  ├─ 解析 → var_signal
│  └─ If var_signal ?= '2' → MoveL 下一拍照姿态
│
├─ Server(发送数据: 3)                    // 臂→PC：确认入账
│
├─ Server(action_signal := 接收到的值)    // 等待 PC 动作信号
├─ 解析 → var_action                    // 支持多位数，如信号10
│
├─ If var_action 包含 '8'               // ── 分支A：放暂存台 ──
│  ├─ Server(接收 8056 暂存坐标行)        // 格式: seq,drop
│  ├─ 解析 seq / pick / drop
│  ├─ MoveJ/MoveL  暂存抓取过渡路点
│  ├─ MoveL  pick
│  ├─ 夹爪闭合
│  ├─ MoveL  drop
│  ├─ 夹爪张开
│  ├─ Server(等待信号10)                   // PC：可继续抓下一件
│  └─ 回到循环顶部（不发0，直接回原料区）
│
├─ ElseIf var_action 包含 '6' 或 '7'    // ── 分支B：直装 ──
│  ├─ If var_action 包含 '7' → 执行长短边转换姿态
│  ├─ 从 box_drop[当前seq] 取箱内点位
│  ├─ MoveL  箱内放置
│  ├─ 夹爪张开
│  │
│  ├─ Server(maybe := 接收)               // PC 可能紧接着发 9（回取暂存）
│  ├─ If maybe 包含 '9'                   // ── 分支B2：从暂存回取 ──
│  │  ├─ Server(接收 8056 暂存槽位坐标)
│  │  ├─ MoveL  暂存 pick → 夹爪闭合
│  │  ├─ 从 box_drop[seq] 放箱
│  │  ├─ Server(等待 6/7 位姿信号)        // 若 PC 单独发
│  │  └─ Server(等待信号10)
│  │
│  └─ Else
│     └─ Server(等待信号10)
│
└─ MoveJ  原料区_扫码路点                  // 信号10后，进入下一件
```

## 信号说明

| 值 | 名称 | 方向 | 含义 |
|----|------|------|------|
| 6 | 直装位姿 | PC→臂 | 按 8055 序号直装，无需换边 |
| 7 | 长短边位姿 | PC→臂 | 直装前需长短边转换 |
| 8 | 放暂存台 | PC→臂 | 将当前抓取物放到暂存台（先收 8056） |
| 9 | 从暂存取 | PC→臂 | 从暂存台指定槽位取出并装入箱子（先收 8056） |
| 10 | 继续抓取 | PC→臂 | 暂存/回取链结束，去原料区抓下一件 |

## 时序约束

1. PC 必须先通过 **8056 发坐标**，再通过 **10000 发信号 8 或 9**
2. `box_drop[seq]` 在批次初从 8055 一次载入；8056 只负责暂存台平铺槽位
3. 接收信号用**字符串包含判断**：先 `'1'` 后 `'2'`，信号 10 判 `'10'`，不用 `split('\n')[-1]`

## ABC / BAC 对照示例

| 步骤 | 抓取 | 扫码 | PC 判断 | 8056 | 信号 | 机械臂动作 |
|------|------|------|---------|------|------|------------|
| 1 | B | B→顺序2 | 2>期望1 | 槽位2坐标 | 8 | B 平铺入暂存台 |
| 2 | A | A→顺序1 | 1==期望1 | - | 6/7 | A 直装；期望→2 |
| 2b | - | - | 槽位2有B | 槽位2坐标 | 9+6/7 | 从暂存取B装箱；期望→3 |
| 3 | C | C→顺序3 | 3==期望3 | - | 6/7 | C 直装；批次完成等信号4 |

## box_drop 数组

```
// 批次开始时（8055）
box_drop[1] = 第一件箱内下放点位
box_drop[2] = 第二件箱内下放点位
...
box_drop[N] = 第N件箱内下放点位

// 直装或从暂存回取装箱时
target = box_drop[seq]
```

`seq` 与 PC 端装箱顺序一致，从 1 开始索引。
